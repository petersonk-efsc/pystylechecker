Python Guidelines

The online Python Style Checker runs python code through two industry standard Python style checkers:
  • pylint (version 2.8.3)
    • pylint Home Page
  • pycodestyle (version 2.7.0) checks code against PEP 8 style conventions
    • pycodestyle Home Page
    • PEP 8 style guide
As an alternative to the online version, you can install both pylint and pycodestyle locally. To install the specific versions used in the online version (and during grading), use the following commands at the command prompt
  • pip install pylint==2.8.3
  • pip install pycodestyle==2.7.0
If you want the latest versions, you may leave off the == part
  • pip install pylint
  • pip install pycodestyle
Note that the pip command was installed when you installed python. If you have trouble running pip, then you may not have added python to your path during installation. At the command prompt, type:
  • python --version
You should see something like:
  • Python 3.8.3
You will want at least version 3.8.3

Python Style Grade Calculation

Some issues are only caught by pylint, others by pycodestyle. Many are caught by both.
Each pylint/pycodestyle issus reduces your final grade.
  • (Twice the number of lines - pylint issues - pycodestyle issues) / (Twice the number of lines)
For example:
  • Your .py file has 100 lines. Use 2 * 100 in the calculation below
  • There are 7 pylint issues
  • There are 3 pycodestyle issues
  • Grade is [(200 - 7 - 3) / 200] = 0.95 = 95%
      Missing/blank files also affect the grade. Each file contributes an equal percent to the final grade. If there are 2 files, then each one is worth 50% of the grade.
